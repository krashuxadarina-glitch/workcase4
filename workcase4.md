## **1\. Робота з менеджерами пакетів у Linux**

У Linux керування програмами здійснюється за допомогою менеджерів пакетів. Це дозволяє централізовано встановлювати, оновлювати та видаляти софт, не шукаючи інсталятори на різних сайтах.

### 1.1 Поняття «пакет» та «репозиторій»

* Пакет (Package) — це архівний файл (у Fedora має формат .rpm), який містить виконувані файли програми, необхідні бібліотеки, скрипти встановлення та метадані (версія, опис, залежності).  
* Репозиторій (Repository) — це віддалене мережеве сховище, де зберігаються тисячі пакетів. Менеджер пакетів звертається до нього, щоб завантажити останню версію програми.

  ### 1.2 Огляд менеджерів пакетів

* DNF (Dandified YUM) — сучасний менеджер пакетів, що використовується у Fedora, RHEL та CentOS. Він вирізняється високою швидкістю роботи та розумним вирішенням конфліктів залежностей.  
* APT — стандарт для систем на базі Debian та Ubuntu.  
* Pacman — надшвидкий менеджер пакетів для Arch Linux.

### **Conclusion**

In this Work-Case, I explored the principles of software management in the Fedora 43 operating system. I gained practical experience using the DNF (Dandified YUM) package manager to perform essential tasks such as searching, installing, and removing software via the command-line interface.

The most valuable part of the work was learning how to handle real-world system challenges. I encountered and successfully diagnosed issues like checksum errors in the graphical store and insufficient disk space in Live mode. By using terminal commands like dnf clean all and df \-h, I optimized the system and completed the tasks by choosing alternative, lightweight software. 

This work proved that the terminal is a more flexible and powerful tool for system administration than a graphical interface.
